## 摘要
视觉推理需要整合分布在区域、属性和关系中的证据，这使得单链推理容易陷入早期感知承诺和幻觉。我们提出Visual Para-Thinker++，一种单策略多智能体框架，其中共享的MLLM策略被实例化为角色条件的主智能体、工作智能体和总结智能体。主智能体通过固定的分配模式分解任务；工作智能体在上下文隔离下并行推理；总结智能体协调完整的工作智能体推理轨迹，而非对最终标签进行多数投票。共享策略通过多智能体能力注入和角色解耦的多智能体优化进行训练，这些方法为相应的token片段分配特定角色的奖励和优势，以减少协作角色间的梯度冲突。一种原生推理引擎通过共享视觉前缀和KV缓存重用实现高效的多智能体展开。在V*、CountBench、RefCOCO系列和HallusionBench上，Visual Para-Thinker++始终优于单轨迹和推理时间并行基线，尤其是在幻觉敏感的视觉推理上表现突出。

## Abstract
Visual reasoning requires integrating evidence distributed across regions, attributes, and relations, making single-chain reasoning prone to early perceptual commitment and hallucination. We propose Visual Para-Thinker++, a single-policy multi-agent framework in which one shared MLLM policy is instantiated as role-conditioned Main, Worker, and Summary Agents. The Main Agent decomposes the task with fixed allocation patterns; Worker Agents reason in parallel under context isolation; and the Summary Agent reconciles full Worker reasoning traces rather than majority-voting on final labels. The shared policy is trained by Multi-Agent Capability Injection and Role-Decoupled Multi-Agent Optimization, which assign role-specific rewards and advantages to corresponding token segments to reduce gradient conflict among collaborative roles. A native inference engine enables efficient multi-agent rollout through shared visual prefix and KV cache reuse. Across V*, CountBench, the RefCOCO family, and HallusionBench, Visual Para-Thinker++ consistently outperforms single-trajectory and inference-time parallel baselines, with especially strong gains on hallucination-sensitive visual reasoning.

## 1. 核心问题与整体含义（研究动机与背景）

- **问题**：视觉推理需要整合分散在图像不同区域、属性和关系中的证据，但现有的单链思维（single-chain-of-thought）方法容易在早期做出感知承诺（perceptual commitment），并且随着推理链延长，错误会不断被巩固而非纠正，导致幻觉（hallucination）问题严重。
- **动机**：单纯延长单条推理链（test-time scaling）在视觉任务中收益有限，因为视觉证据是分布式而非集中式的。作者提出一种新的扩展轴：将视觉推理组织为多角色协作，让不同智能体分别承担问题分解、局部证据收集、全局验证、矛盾调和等不同角色，从而减少单一轨迹的早期错误固化。
- **整体含义**：通过在单个共享策略中实例化多个角色智能体（Main, Worker, Summary），实现多角度并行推理与轨迹级整合，同时通过角色解耦优化避免梯度冲突，在多个视觉推理基准上显著优于单轨迹和并行基线方法。

## 2. 方法论：核心思想、关键技术细节

### 2.1 核心思想
- **单策略多智能体框架（Single-Policy Multi-Agent）**：一个共享的MLLM策略（Qwen2.5-VL）通过角色token条件（role conditioning）实例化三个角色智能体——主智能体（Main Agent）、工作智能体（Worker Agents）、总结智能体（Summary Agent）。所有智能体共享同一套网络权重，角色身份仅由角色token和可见上下文决定。
- **固定四路径协作协议**：主智能体将任务分解为子问题并分配给四个工作智能体；工作智能体在上下文隔离（context isolation）下并行推理；总结智能体读取所有工作智能体的完整推理轨迹并产生最终答案（而非多数投票）。

### 2.2 关键技术细节

#### (a) 主智能体任务分配模式
- **Block-based（块式）**：将图像划分为K个不相交区域（默认四象限），每个工作智能体收集局部证据。适用于精细感知、属性和接地任务。
- **Scan-order（扫描顺序）**：每个工作智能体拥有全局视野，但按不同扫描顺序遍历图像（如从左到右、从右到左、从上到下、从下到上）。适用于计数和全局验证。

#### (b) 多智能体能力注入（Multi-Agent Capability Injection, MAC）
- **第一阶段监督微调**：使用更强MLLM教师合成的多智能体轨迹（包含主、四个工作、总结共K+2个角色段）进行训练。
- **上下文隔离掩码（Context-Isolation Mask）**：在打包轨迹中，每个工作智能体只关注主智能体输出和自己生成的token，不看到其他工作智能体；总结智能体拥有全部可见性。防止工作智能体相互复制，促进互补探索。

#### (c) 角色解耦多智能体优化（Role-Decoupled Multi-Agent Optimization）
- **奖励设计**：
  - **Worker奖励** $R^{(i)}_{\text{worker}} \in [0,1]$：基于跨工作智能体多数投票启发式方法，评估局部推理质量。
  - **结果奖励** $R_{\text{out}}(\tau) \in \{0,1\}$：总结智能体最终答案是否匹配真实标签。
- **优势分解**：对每个奖励源进行组归一化，得到 $A_{\text{out}}$ 和 $A^{(i)}_{\text{worker}}$。然后按token段组合：
  - Main和Summary token：$A_t = A_{\text{out}}$
  - Worker i token：$A_t = A_{\text{out}} + \lambda A^{(i)}_{\text{worker}}$（$\lambda=0.5$）
- **损失函数**：使用DAPO（on-policy）变体的截断代理损失，优势按上述方式按角色分配，实现梯度掩码（gradient mask），避免Worker和Main/Summary梯度冲突。

#### (d) 原生多智能体推理引擎
- 基于vLLM实现，使用PagedAttention Copy-on-Write机制：
  - 图像-问题-Main段预填充一次，共享KV缓存。
  - 在角色触发token处fork出K个工作序列，并行解码。
  - 合并所有Worker KV页面后交给Summary段，避免冗余预填充。

## 6. 主要结论与发现

1. **单策略多智能体框架有效提升视觉推理**：通过角色解耦的并行推理和轨迹级整合，显著优于单链CoT和多数投票等并行方法。
2. **角色解耦优化是关键**：将Worker奖励和结果奖励在优势级别组合（式4）而非简单求和，避免了梯度耦合，是性能提升的主要来源。
3. **Worker奖励和角色解耦优势相互补充**：两者同时开启效果最佳。
4. **适用任务类型**：在幻觉敏感、计数、精细感知任务上增益最大（如Pixmo-test +17.9, V* +16.7, HallusionBench +7.9）。
5. **效率可行**：通过KV缓存复用和并行解码，开销远低于独立多模型方案。

## 7. 优点

- **概念创新**：提出“单策略多智能体”范式，用一个共享策略实现多角色协作，避免了多模型部署的高成本。
- **方法优雅**：简单的角色token和上下文隔离掩码即实现角色分离，无需独立模型或复杂通信协议。
- **优化设计**：角色解耦优势组合（式4）巧妙地通过token级梯度掩码解决多角色梯度冲突，是核心贡献。
- **工程扎实**：原生推理引擎利用PagedAttention优化，将多智能体推理的实际开销降低到接近单模型水平。
- **实验充分**：覆盖多种视觉任务，消融实验系统，验证了每个设计选择的作用。

## 8. 不足与局限

- **Backbone规模有限**：主要实验在3B上进行，7B仅作为补充（附录A），未在更大模型（如30B+）上验证，扩展性未知。
- **角色组织固定**：当前主智能体使用两种固定分配模式（块式/扫描顺序），未实现自适应、自由规划，灵活性受限。
- **Worker奖励设计脆弱**：基于多数投票启发式，当大多数Worker路径共享相同幻觉时可能强化错误。
- **Summary奖励仅依赖最终答案**：未显式监督总结智能体的调和过程是否忠实，仅靠结果奖励。
- **计算成本仍随Worker数量增加**：虽然优化了，但K=4是最优甜点，K=8时增益极小而成本翻倍。
- **未消融Summary智能体自身决策**：未区分“阅读轨迹”和“调和证据”的贡献（例如，用非学习性多数投票代替总结智能体会怎样）。
- **评估域覆盖有限**：未包含文档理解、图表推理、视频决策等更广泛的多模态场景。
- **伦理风险**：增强的推理能力可能被用于生成更令人信服的错误分析；训练数据可能继承教师偏差。
