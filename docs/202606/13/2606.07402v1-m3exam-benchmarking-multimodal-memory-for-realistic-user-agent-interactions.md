## 摘要
语言智能体越来越多地被部署在积累多模态信息的环境中，但现有基准测试假设的是人类-人类形式，具有稀疏的视觉内容和直接的内容，既不评估对真实多模态文件交互的推理，也不评估对隐藏用户信息的解读。因此，我们引入M$^3$Exam，这是一个基于真实用户-智能体交互的以查询为中心的多模态对话记忆基准测试，具有跨模态基础推理和隐式信息推断的多维评估。对多模态大语言模型和记忆系统的基准测试揭示了跨模态基础推理、跨会话推理以及积累多模态上下文的效率成本方面的持续差距。我们进一步提出M$^3$Proctor，一种多模态记忆方法，检测查询模态偏差并仅按需消耗原始视觉源，将准确率提高13%，同时将索引构建时间和检索的标记数量减少超过70%。

## Abstract
Language agents are increasingly deployed over accumulating multimodal information, yet existing benchmarks assume a human-human form with sparse visuals and straightforward content, evaluating neither reasoning over authentic multimodal file interaction nor the interpretation of concealed user information. We therefore introduce M$^3$Exam, a query-centric multimodal conversational memory benchmark built on realistic user-agent interaction, with multi-dimensional evaluation spanning cross-modal grounding and implicit information inference. Benchmarking MLLMs and memory systems reveals persistent gaps in cross-modal grounding, cross session reasoning, and the efficiency cost of accumulating multimodal context. We further propose M$^3$Proctor, a multimodal memory method that detects query modality bias and consumes raw visual sources only on demand, improving accuracy by 13% while cutting index-construction time and retrieved tokens by over 70%.

## 2. 论文提出的方法论

### 2.1 基准设计（M³Exam）

- **核心思想**：构建真实的用户-智能体多轮交互场景，每个实例包含一段多会话对话历史 \( \mathcal{D} = (D_1, ..., D_L) \) 和一个查询 \( x \)。智能体需要从历史中检索相关证据并生成答案。
- **构建流程**：
  1. **输入源**：每个实例由人物角色（Persona）、核心事件（Core Event）和多模态文件池（图像、PDF等）生成。
  2. **时间线构建**：利用 LLM 生成按时间排序的核心事件序列，并加入干扰事件，经自检和人工审查后确认。
  3. **多轮对话合成**：通过滑动窗口生成多轮会话，每轮携带图像/文件，助手回复基于累积记忆。会话经过两阶段质量控制（局部窗口+跨块审核）。
  4. **问题库合成**：生成八种类型的问题（记忆型、推理型、解释型），共 5,150 个问题。每种问题有金标答案和多个渐变替代答案，支持分级评分。
- **关键技术细节**：
  - 问题涵盖检索、多跳推理、时间推理、多模态推理、跨模态文件管理、隐式推断和领域主题推理。
  - 评估指标包括 Exact Match (EM)、F1、BLEU-1、LLM-as-a-Judge，并加权得到 Overall Score。

### 2.2 基线方法（M³Proctor）

- **核心思想**：检测查询的模态偏差（即查询依赖于哪种模态），仅按需消耗原始视觉源，避免盲目注入所有视觉 token 导致的高成本和噪声。
- **算法流程**：
  1. **多模态索引构建**：将图像、PDF 等原始视觉源投影为文本替代物（图像描述、图表转录、PDF 摘要），并附上模态标签。同时构建精简的会话摘要块。
  2. **模态感知推理**：
     - **偏差检测**：对每个查询，通过一次 LLM 调用确定其模态偏差向量 \( b(q) \in \{0,1\}^{|M|} \)（M={图像，PDF，图表}）。
     - **重排序**：基于语义相似度和模态偏差调整候选 chunk 的分数，公式 \( s(q,c) = \cos(q,c) + \sum \lambda_m b_m(q) \mathbb{1}_m(c) \)，并加入多样性约束。
     - **代价感知级联**：先尝试仅用文本替代品回答；若置信度低或模态偏差不满足（通过置信度函数 \( \Phi \) 和融合证据分数 \( e_m(q) \) 判断），再级联原始视觉源。
- **效果**：准确率提升 13%，索引构建时间减少 80%，检索 token 减少 70% 以上。

## 6. 论文的主要结论与发现

- **M³Exam 对当前系统极具挑战**：最强前沿 MLLM（GLM-5.1）仅达到 0.549 Overall Score，即使 Oracle 设置下也远未解决。
- **瓶颈是跨模态基础和隐式意图推理**：模型在单会话检索中表现尚可，但在多模态推理（MR）、隐式推断（II）、主题推理（TH）等类别上大幅下降。外部记忆系统可部分缓解，但多数仍不足。
- **M³Proctor 有效**：在 7B 骨干上取得最佳 Overall Score（0.484），超越所有现有记忆系统，甚至优于部分更大规模的前沿 MLLM。其模态偏差检测和级联策略在保持准确率的同时大幅降低开销（索引构建时间 72 秒，每查询 token 数 4,591，远低于多模态基线）。
- **多模态依赖性**：基准问题高度依赖视觉证据，M³Proctor 的级联在视觉类型问题上触发频率高（68% 在 FM，51% 在 MR），且触发后带来显著收益（+0.116 在 FM，+0.092 在 MR）。

## 7. 优点

- **基准设计的真实性**：采用合成的多会话对话、多样角色、真实文件（PDF、图表、照片），并包含隐式推断这一现有基准缺失的维度。

- **分级评估指标**：不仅提供二值判断，还通过渐变答案列表和 LLM-as-a-Judge 实现细粒度评分。
- **高效的方法设计**：M³Proctor 的模态偏差检测仅需一次文本 LLM 调用，级联代价可控，在保持高准确率的同时显著降低资源开销，具有实用价值。
- **全面的实验对比**：涵盖多种骨干、多类记忆系统，并提供消融和效率分析，结论可靠。

## 8. 不足与局限

- **单轮 QA 限制**：当前基准主要评估累积历史上的单轮问答，未涉及真实交互中常见的多轮意图演进和迭代记忆更新。作者承认计划未来扩展至更动态的多轮记忆评估。
