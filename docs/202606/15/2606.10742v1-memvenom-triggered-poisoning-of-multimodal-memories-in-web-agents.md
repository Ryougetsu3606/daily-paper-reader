## 摘要
外部记忆已成为现代Web Agent的核心组件，通过检索过往经验实现长期推理。然而，这一范式引入了一个关键漏洞：注入记忆中的恶意内容能被持久召回并反复影响Agent行为。在本工作中，我们识别并系统研究了多模态记忆投毒——Web Agent系统中一个被忽视却切实存在的攻击面。我们提出MemVenom，一个统一的黑盒攻击框架，利用图文协同证据污染图结构的外部记忆。我们的方法包含两阶段设计：（1）触发条件检索攻击，确保恶意记忆以高概率被召回；（2）检索后攻击诱导，利用对抗扰动与隐蔽OCR注入覆盖原始用户目标。与以往针对提示或纯文本记忆的攻击不同，我们的方法无需修改模型参数或重新优化恶意任务，即可实现持久、可复用且目标无关的攻击。跨多个Web Agent框架与视觉语言模型的实验表明，MemVenom在最小的良性性能影响下实现了强大的端到端攻击成功率，在GPT-5系列Web Agent上高达99.15%，并能有效迁移至不同架构与模型规模。

## Abstract
External memory has become a core component of modern web agents, enabling long-horizon reasoning through the retrieval of past experiences. However, this paradigm introduces a critical vulnerability: malicious content injected into memory can be persistently recalled and repeatedly influence agent behavior. In this work, we identify and systematically study multimodal memory poisoning, an overlooked yet practical attack surface in web-agent systems. We propose MemVenom, a unified black-box attack framework that poisons graph-structured external memory with coordinated text-image evidence. Our method consists of a two-stage design: (1) a trigger-conditioned retrieval attack that ensures high-probability recall of malicious memory, and (2) a post-retrieval attack induction that leverages adversarial perturbations and stealthy OCR injection to override the original user objective. Unlike prior attacks that operate on prompts or text-only memory, our approach enables persistent, reusable, and goal-agnostic attacks without modifying model parameters or re-optimizing malicious tasks. Experiments across multiple web-agent frameworks and vision-language models demonstrate that MemVenom achieves strong end-to-end attack success with minimal impact on benign performance, reaching up to 99.15% on GPT-5-family web agents, while transferring effectively across architectures and model scales.

- **核心问题**：现有安全研究集中在即时提示注入、越狱攻击或纯文本检索投毒，缺乏对多模态（图文）记忆投毒在 Web Agent 场景下的系统性研究。本文旨在揭示这种持久性、可复用的攻击面。

### 2. 方法论
- **核心思想**：设计两阶段联合优化框架，同时提升恶意记忆被召回的概率和召回后对 Agent 行为的主导能力，且支持目标模块化（替换目标无需重新优化触发器和后召回组件）。
- **关键技术细节**：
  - **触发条件检索攻击**：优化一个视觉触发器 τ，使带触发器的网页截图在嵌入空间形成紧凑且远离良性记忆空间的簇，确保触发条件下恶意子图高概率被召回。优化目标包括簇紧凑性、与良性观测分离、与良性记忆空间低重叠三部分。
  - **后召回复合视觉攻击**：构建一个可复用的记忆优先级组件，结合 ℓ∞ 受限的对抗扰动和隐蔽 OCR 注入。对抗扰动使扰动后的基图与桥接图像及优先级指令在特征空间对齐；OCR 注入以隐式文本线索强化优先级信号。优先级组件本身不绑定具体恶意目标，可跨目标复用。
  - **恶意子图组装**：将召回组件、优先级组件和可替换的目标组件通过图边连接，注入到干净记忆图中，形成触发条件下可激活的恶意子图。
- **公式/算法流程**（文字描述）：
  1. 收集干净截图集，对触发器 τ 优化嵌入紧凑性和分离性，得到最优触发器 τ⋆ 和代表截图 x⋆_ret，构建召回节点 v⋆_ret。

### 6. 主要结论与发现
- MemVenom 在三个 Web Agent 框架和多个 VLM 上均取得高攻击成功率，端到端 ASR-ra 最高达 99.15%（SeeAct + Qwen3-VL-PLUS），且性能稳定。
- 对良性任务影响小，PU 与 BU 接近，表明攻击不破坏正常功能，仅在触发条件下激活。
- 显著优于改编的基线方法，后者在恶意召回或后召回诱导上存在明显短板。
- 检索器迁移性实验显示优化触发器可跨不同检索器有效，不依赖单一编码器。
- 现有轻量级防御（安全规则、LlamaGuard）无法完全防御此类攻击，需更复杂的记忆溯源与任务对齐检查。

### 7. 优点
- **威胁模型现实性**：黑盒设定，仅需记忆写入权限，无需访问 VLM、检索器或梯度，符合实际部署场景。
- **持久性与可复用性**：攻击效果不随会话结束而消失，恶意子图可重复利用，且目标组件可独立替换，无需重新优化。
- **目标无关性**：优先级组件不绑定具体恶意目标，实现“一次优化，多次使用”。
- **泛化能力**：跨多种 Agent 框架、VLM 规模、检索器类型表现稳定。
- **隐蔽性**：通过对抗扰动和微小 OCR 注入使得图像变化难以察觉，且非触发条件下恶意记忆很少被召回。

### 8. 不足与局限
- **实验环境受限**：所有评估均在受控沙盒中进行，未部署到真实生产环境；实际记忆中可能涉及更多异质的管道、检索策略和行动约束。
- **记忆类型单一**：仅考虑了基于截图的图结构多模态记忆，未覆盖工具特定记忆、用户画像记忆、动态更新情景记忆等其他设计。
- **防御研究初步**：仅测试了两种轻量级即插即用防御，尚未探索更复杂的记忆溯源、检索一致性、任务对齐与行动后果联合推理的防御方案。
- **触发依赖性**：攻击需要攻击者能够预先注入记忆并掌握观测中出现触发器的时机，在完全随机检索场景下可能受限。
