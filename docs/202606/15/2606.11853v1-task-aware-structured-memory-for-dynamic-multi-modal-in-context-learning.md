## 摘要
多模态大语言模型（MLLMs）依赖上下文学习（ICL）实现快速任务适应，但其可扩展性受到有限上下文窗口以及长多模态序列中键值（KV）缓存成本增长的严重限制。现有的记忆压缩方法通常依赖于刚性的 token 移除或基于样本的重要性估计，这引入了偏差，破坏了语义结构（尤其是对于视觉表示），并产生无法适应新查询的静态记忆。我们提出了 TASM（任务感知结构化记忆），这是一个无需训练的框架，通过任务感知、结构保持和动态可访问的记忆构建来应对这些限制。TASM 采用任务向量引导压缩，用捕捉示例间共享相关性的任务级方向替代样本特定信号。为了保持底层流形，它通过二分图匹配进行语义感知的 token 合并，聚合 token 而不进行破坏性剪枝。最后，TASM 将记忆结构化为一个层次结构，包括紧凑的核心记忆（Core Memory）和潜在库（Latent Bank），支持查询自适应的动态检索。评估证实 TASM 在重度压缩下仍保持高性能，有效平衡了效率与适应性。

## Abstract
Multi-modal large language models (MLLMs) depend on in-context learning (ICL) for rapid task adaptation, but their scalability is severely limited by finite context windows and the growing cost of key-value (KV) caches in long multi-modal sequences. Existing memory compression approaches typically rely on rigid token removal or sample-dependent importance estimation, which introduces bias, disrupts semantic structure, particularly for visual representations, and yields static memories that cannot adapt to new queries. We introduce TASM (Task-Aware Structured Memory), a training-free framework that addresses these limitations through task-aware, structure-preserving, and dynamically accessible memory construction. TASM employs task-vector guided compression to replace sample-specific signals with a task-level direction that captures shared relevance across demonstrations. To preserve the underlying manifold, it applies semantics-aware token merging via bipartite graph matching, aggregating tokens without destructive pruning. Finally, TASM structures memory into a hierarchy comprising a compact Core Memory and a Latent Bank, facilitating query-adaptive dynamic retrieval. Evaluations confirm TASM maintains high performance under heavy compression, effectively balancing efficiency with adaptability.

## 一、核心问题与整体含义（研究动机与背景）

- **背景**：多模态大语言模型（MLLMs）通过上下文学习（ICL）实现快速任务适应，无需参数更新。但“Many‑Shot”场景（使用成百上千个演示样本）带来了巨大的计算开销——长序列中 Key‑Value（KV）缓存迅速膨胀，超出 GPU 显存并增加延迟。
- **核心问题**：现有缓存压缩方法（如基于注意力分数的硬剪枝）存在三个结构性缺陷：
  1. **样本特定偏差**：依赖答案感知的注意力分数，过度拟合单次演示，忽略全局任务共性。
  2. **拓扑结构破坏**：对视觉 token 进行硬剪枝（Top‑K 删除）会破坏 2D 空间流形，导致定位、空间推理等任务准确率骤降。
  3. **静态记忆刚性**：压缩后的缓存变为静态，无法在遇到需要细粒度细节的新查询时恢复被丢弃的信息。
- **整体目标**：提出一种**训练无关**、**结构保持**且**查询自适应**的记忆压缩框架，使 MLLMs 能在长上下文中高效且准确地执行 ICL。

## 二、方法论：核心思想与关键技术细节

### 2.1 总体框架（TASM）
TASM 将 KV 缓存视为“任务的潜在结构表示”，通过三个耦合的模块实现压缩与动态访问，全程无需参数更新。

### 2.2 任务向量引导的重要性估计（Task‑Vector Guided Importance）
- **任务向量提取**：从少量演示的问答对中，分别计算问题表示与答案表示的语义质心，取归一化差作为任务向量 \(\tau_l\)，代表“从问题到答案”的推理方向。
- **正交投影评分**：将每个 token 的键表示投影到任务向量 \(\tau_l\) 上，得分 \(s_{i,l}^{\text{task}} = \text{ReLU}\left( \frac{k_{i,l}^\top \tau_l}{\|k_{i,l}\|} \right) + \gamma \frac{\|v_{i,l}\|_2}{\max_j \|v_{j,l}\|_2}\)。只有与任务方向正对齐的 token 被保留，去除 attention 下沉等噪声。
- **层自适应门控**：浅层更多依赖局部注意力分数（保留空间/语法特征），深层更多依赖任务分数（保留语义），通过移位 sigmoid 函数 \(\lambda(l)\) 平滑过渡。

### 2.3 语义感知的 Token 合并（Semantics‑Aware Token Merging）
- **图构造**：将 token 分为高分的“汇节点”（保留）和低分的“源节点”（待合并），构建二分图。
- **空间约束兼容性**：对视觉 token 强制局部窗口约束（默认 \(\Delta_{\text{win}}=3\)），防止跨空间远距离合并破坏 2D 坐标结构。
- **流形保持聚合**：通过最优传输（贪心匹配）将源节点合并到最近的汇节点中，汇节点更新为加权质心，实现“软压缩”而非硬删除。

### 2.4 信息几何动态检索（Information‑Geometric Dynamic Retrieval）
- **层次记忆**：将压缩后的核心记忆（Core Memory）驻留 GPU，其余 token 存入 CPU 的潜在库（Latent Bank）。
- **触发条件**：计算当前查询注意力分布与前一步分布之间的 Jensen‑Shannon 散度 \(D_{\text{JS}}\)。当超过阈值 \(\delta\) 时，表示语义偏移，触发检索。
- **稀疏检索**：从潜在库中取出与当前查询最相关的 top‑k 个 token，临时扩展核心记忆。平均复杂度仍为 O(1)，仅 3–4% 的步骤需要检索。

## 三、实验设计

### 3.1 数据集与 Benchmark
- **通用视觉识别**：ImageNet‑100
- **空间/定位**：ScreenSpot（GUI 元素定位）、RefCOCOg（引用表达定位）
- **复杂推理/知识**：OK‑VQA、MME‑RealWorld（MME‑RW）
- **时序/视频**：YouCook2（视频描述）、VideoMME（长视频理解，最长达 30 分钟以上）
- **细粒度检索压力测试**：Visual Needle‑in‑a‑Haystack（V‑NIAH，从大量图像中找回特定细节）

### 3.2 对比方法
- **全上下文基线**：MLoC（直接使用全部演示）
- **现有压缩方法**：EMLoC（SOTA 多模态压缩）、SnapKV、FastV、SparseVLM（均为基于注意力或硬剪枝的方法）
- **基础模型**：Qwen2‑VL‑7B‑Instruct（主要），并在 Qwen2.5‑VL‑7B、LLaVA‑NeXT‑Video（7B）、InternVL3（8B）上验证泛化性。

### 3.3 实验设置
- 演示数：通常 200 个示例（部分场景 50 个），并变化 0–300 个观察缩放规律。
- 压缩比：核心记忆保留 20% token，潜在库 40%；最终上下文长度仅为全上下文的 15–20%。

## 六、主要结论与发现

1. **性能**：TASM 在 **79–85% 压缩率**下，在所有 9 个基准上**持平或超越**全上下文 MLoC，并显著优于 EMLoC。
2. **空间保留**：在 ScreenSpot、RefCOCOg 等定位任务上，TASM 的语义合并比 EMLoC 的硬剪枝平均高 **3–5 个百分点**，证明拓扑结构未被破坏。
3. **动态检索有效**：在 V‑NIAH 中，当 haystack 含 200 张图时，TASM 准确率 45.5%，EMLoC 仅 18.9%，证明潜在库加 JS 触发能防止信息“永久丢失”。
4. **缩放行为**：在 ImageNet‑100 上，TASM 随演示数增加保持对数线性增长（300 示例时 66.8%），EMLoC 在 200 示例后出现饱和下降。
5. **架构通用性**：对动态分辨率模型（InternVL3）同样有效，避免因 tile 丢失导致的大幅退化（SnapKV 降 8.3 点，TASM 仅降 0.4 点）。
6. **效率**：解码速度保持 43 tok/s 以上，几乎与 EMLoC 持平，但推理召回更精准。

## 七、方法优点

- **训练免费**：冻结模型参数，不需要额外的微调或蒸馏，即插即用。
- **结构保持**：用语义合并替代硬剪枝，保留视觉 token 的 2D 流形和空间关系，对定位任务特别重要。
- **动态自适应**：基于注意力分布扰动（JS 散度）的动态检索，只在需要时调用潜在库，避免静态策略的盲目开销。
- **任务感知**：利用任务向量提取通用推理方向，过滤样本特异性噪声，并且可零样本迁移到同类新任务。
- **强鲁棒性**：对不同模型架构、不同分辨率策略均表现出稳定增益。

## 八、不足与局限

1. **任务向量依赖**：需要少量（≧4 个）演示才能稳定提取优质向量。在极冷启场景下，若可用的演示样本极少或噪声极大，任务向量的质量可能下降。
2. **实验规模有限**：所有实验均在 **7B‑8B** 规模的 MLLM 上进行。更大模型（>70B）上的效果、KV 缓存扩展性尚未验证。
3. **未考虑多轮对话**：当前场景仅为单次查询 + 演示集。在交互式、递增式任务（如持续更新记忆的智能体）中，记忆的插入与淘汰策略需要额外研究。
4. **长期时序任务覆盖**：虽然 VideoMME 最长 30 分钟，但未测试数十小时的长视频或无限流式场景。潜在库的容量和检索效率在极端长度下可能成为瓶颈。
5. **潜在偏差风险**：任务向量提取仍依赖于少数演示的问答对。若演示存在标注偏差或分布偏移，压缩可能强化错误模式。
6. **超参数微调**：文中虽然给出默认值（空间窗口 3、JS 阈值 0.002、插值权重 α=0.3），但在不同模态分布或数据特点下可能需要调参，泛化性有待更多验证。
